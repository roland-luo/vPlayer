## M1 — 最小可运行播放器

**目标：** 完成 Tauri + libmpv 的骨架集成，能打开视频文件、渲染画面、基础控制。

### 核心交付

| 模块 | 文件 | 说明 |
|------|------|------|
| MPV 核心 | `src/mpv/core.rs` | mpv 实例生命周期（创建 / 初始化 / 销毁） |
| MPV 渲染 | `src/mpv/renderer.rs` | OpenGL 渲染上下文管理，帧输出到纹理 ★ |
| MPV 事件 | `src/mpv/event.rs` | libmpv 事件循环 → Tauri 事件转换 |
| IPC 层 | `src/ipc/commands.rs`, `src/ipc/events.rs` | 前后端通信链路（invoke / event） |
| 前端 MVP | `frontend/src/App.vue` 等 | 播放界面、控制栏、进度条 |

### M1 关键风险与对策

| 风险 | 对策 | 负责人 |
|------|------|--------|
| libmpv 初始化失败（GPU 不支持、虚拟机环境） | 启动回退策略：检测失败 → 显示友好错误页面 → 引导检查驱动 | 待分配 |
| OpenGL 纹理分配失败（窗口 resize、显存不足） | `render_frame()` 中捕获错误 → 向前端发送 `video:error` → 显示错误遮罩 | 待分配 |

### M1 阻塞项

- `mpv_create()` 或 OpenGL 上下文初始化失败时，需要前端独立的错误页面组件
- `src/mpv/renderer.rs` 完成后，才能接入纹理分配失败的错误处理

## M2 — 播放体验完善

**目标：** 补齐日常播放所需功能，达到可用状态。

### 核心交付

| 模块 | 说明 |
|------|------|
| 播放列表 | 打开本地文件/文件夹、列表持久化、拖拽排序、记忆上次列表 |
| 字幕支持 | 内嵌字幕检测、外挂字幕加载、字体配置、样式调整 |
| 音轨切换 | 多音轨检测、切换、偏好语言记忆 |
| 设置持久化 | 播放器偏好（音量、倍速、窗口大小、上次播放位置）保存到本地配置 |
| 错误提示 | 文件损坏、编码不支持、网络失败等场景的用户可见提示 |

### M2 依赖

- M1 完成（基础播放链路已跑通）

---

## M3 — 插件系统 MVP

**目标：** 引入可扩展的插件机制，加载并隔离第三方代码。

### 核心交付

| 模块 | 文件 | 说明 |
|------|------|------|
| 命令总线 | `src/plugin/bus.rs` | `emit()` / `invoke()` + 错误边界隔离 ★ |
| 插件加载器 | `src/plugin/loader.rs` | 插件加载与生命周期管理 |
| 沙箱 | `src/plugin/sandbox.rs` | 插件权限与隔离机制 |
| 注册表 | `src/plugin/registry.rs` | 插件注册表 |
| 内置插件 | 2–3 个官方示例 | 如截图、快进跳过片头、自动字幕下载 |

### M3 关键设计决策

| 问题 | 当前方向 | 待确认 |
|------|----------|--------|
| 插件崩溃后是否自动禁用？ | 建议：是，并记录日志 | — |
| 是否向用户显示插件崩溃通知？ | 建议：是（非阻断 toast） | — |
| 插件错误日志和上报机制 | 统一输出到 `~/.config/vPlayer/plugin-logs/` | — |

### M3 依赖

- M1 命令总线最小版本完成
- M2 基础播放功能稳定

## M4 — 稳定与优化

**目标：** 提升产品质量，完成多平台适配，准备发布。

### 核心交付

| 模块 | 说明 |
|------|------|
| 遥测埋点 | 启动成功率、崩溃率、播放流畅度、GPU 兼容性统计 |
| 性能调优 | GPU 内存占用优化、大尺寸视频播放优化、启动时间优化 |
| 多平台测试 | Windows（WebView2）、macOS（WKWebView）、Linux（WebKitGTK） |
| 错误回退策略完善 | 启动/渲染/插件各环节的用户引导与降级方案 |
| 文档与发布 | 用户手册、插件开发文档、GitHub Release |

### M4 依赖

- M1–M3 全部完成
